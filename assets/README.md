# assets/

Images referenced from the repository's top-level `README.md` and from documentation. Place the following files here before publishing the repository publicly:

| File              | Purpose                                                  | Suggested Size / Format |
|-------------------|----------------------------------------------------------|-------------------------|
| `logo.png`        | Y2Pal brand logo (used in Topics preview, Sponsor panel) | 512×512 PNG, transparent|
| `product-box.png` | README "Preview" image — workflow diagram or hero visual | 1200×630 PNG            |
| `banner.png`      | Hero banner for top of GitHub repo preview               | 1280×640 PNG            |
| `quality-icon.png`| Small icon illustrating output quality                   | 256×256 PNG             |
| `format-icon.png` | Small icon illustrating output format                    | 256×256 PNG             |

## Naming Notes

- Keep filenames lowercase with hyphens.
- Prefer PNG over JPG for icons and diagrams (lossless compression).
- Use WebP only if all downstream consumers are modern browsers.

## Placeholder Behavior

If these files are missing when the README renders on GitHub, broken-image icons will appear in the Preview section. Either supply real images before publishing or remove the image references from `README.md`.

## Source

The design assets are part of Y2Pal's brand kit. Contact the Y2Pal team for the latest approved versions before publishing.
